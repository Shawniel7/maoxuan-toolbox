# 贡献指南

感谢你愿意给思想工具箱加新条目 / 修正老条目!

## 新增一个思想工具

1. Fork → 新建 branch
2. 编辑 `data/entries.json`,追加一个对象,字段完整(见 [`data/schema.md`](data/schema.md))
3. 本地运行 `python3 -m http.server 8080`,确认:
   - 输入一段能命中你新词条 `trigger_keywords` 的文本,结果正确出现
   - `browse.html` 能列出你的新卡片
   - 原文引文的 URL 确实打得开
4. 提 PR,在描述里说明:
   - 选的是哪一篇原著 / 哪一节
   - 为什么这个工具值得收录(面向的是哪一类困惑)
   - 类比故事的出处(历史事件 / 当代场景)

## 硬要求

- **原文引用 ≤ 80 字**,单句清晰,指向一个具体段落。
- **必须标明出处**(篇名 + 节次 + URL),URL 必须在提 PR 时可访问。
- **优先公开权威源**:
  - <https://www.marxists.org/chinese/maozedong/> · 中文马克思主义文库
  - <https://cpc.people.com.cn/> · 中国共产党新闻网
  - <https://www.12371.cn/> · 共产党员网
- **类比故事必须真实存在**。历史事件要经得起核查,不编细节。如果实在找不到合适的历史案例,用当代场景替代(标注为"当代类比")即可。
- **行动清单必须可执行**。避免"多反思"、"保持积极"这类无法操作的建议;每一条都应该是一个具体动作。
- **不引入政治立场**。项目只处理方法论层面,不讨论具体政治评价。

## 修正现有条目

- 错别字、URL 失效、翻译不准 —— 直接 PR,不必开 issue。
- 对"现代译读"的改写,希望在 PR 里说一下你的出发点:你觉得原版在哪个场景下会被误解?

## 行为准则

- 尊重不同观点。这里面的几个工具(矛盾论、实践论)在学界和大众里都有长期争议,我们只挑出它们作为**分析工具**时稳定有效的那部分。
- 讨论条目时对事不对人。
- 避免把这个项目当政治场域使用 —— 任何明显政治化的 PR 都会被拒。
